Gated command (Measure menu, Frequency Response submenu, for Drivers)

Use this command to measure the gated response of a driver. This command assumes that the time data is gated to the two time markers. Input is assumed to be from a Microphone and any Microphone calibration is used.

The measurement is performed by sending an MLS (Maximum Length Sequence) wave out through the sound board. The source input wave is analyzed and combined with the reference input and the Microphone calibration, and the result is stored in the driver properties.

Change the volume, the length of the sequence, and other settings in the Options Preferences dialog. If you want to switch to the microphone input on the sound board, go to the mixer panel and switch the input there - sorry, there is no programmatic way to do this.

This command is optional.

This command changes the current properties of the driver, inserting a new dataset entry into one slot.

This command is identical to the Measure Frequency Response command, except for where the dataset is placed. It is primarily used as a second storage location for a gated response or when calculating full response of a woofer using nearfield, gated, and (possibly) the port responses.
