Optimize Network dialog (Network, Optimize Network…)

Use this command to have Speaker Workshop attempt to pick the best component values to match a crossover+driver response to a desired goal response. This is not guaranteed to always work, and you may need to fiddle the frequency range / number of points / components to vary options until you get results you really like.

Fields:    
Frequency Range Start Set the lower frequency (for a woofer) to a point where the crossover has some impact, perhaps 1-2 octaves below the crossover frequency.
  End Set the higher frequency (for a woofer) to the point where the desired response is down 20-40dB (say 1-2 octaves above the crossover frequency)
  Number of Points Start with a small number of points (must be more than the number of components), say 10-20 and then increase the # of points as you zero in on a good result.
Target frequency response This is the goal you want the optimizer to match to. Make sure the level of this response is appropriate for the driver output (the driver’s frequency response dataset).
Components to vary You may wish to not have the optimizer act on some of the components, either to keep standard values for capacitors (for example) or to stop it from incorrectly removing impedance matching components. Check the components you DO want to have the optimizer work with.

This command changes the current network. You can undo this.

For best results, we recommend that you examine a chart containing the goal and the crossover response. Recalculate the crossover response after this completes and compare the two.

Also, you may wish to add/subtract components based on the results of the optimize routine.

Comments: this routine only works with a single driver in the network. It also requires just a single generator. The driver must have both a frequency response and an impedance (Driver Properties / Data tab).
