Perturb Components Dialog (Network Perturb Components)

Use this dialog to perform multiple response calculations for the current network while varying a component. The response is based on the impedances of each passive component as well as the measured impedance (Impedance) and response (On Axis) of each driver.

Fields:    
  Component Select a component from the network to vary.
  Start value The starting (low) value for calculations. Suffixes (K,m,u,p,n) are allowed.
  End value The ending (high) value for calculations. Suffixes are allowed.
  Number of points How many different values between start and end (inclusive) will be used. A common value is 5 or 6.

This command is often used to optimize a component within a network.

This command creates or replaces datasets named mynetwork.Freq.value.
